1.2 — Hero Section Minimum Height Fix
Rule — apply to every page's hero section across the entire website:
The hero section must always be at minimum the full viewport height.
css
min-height: 100vh;

Implementation with Tailwind:
text
className="min-h-screen"

Behavior:
If content is less than viewport height — hero still fills full screen (min-h-screen)
If content exceeds viewport height — hero expands naturally to fit content
Apply this rule to hero sections on ALL pages:
Home, Features, Solutions, Integrations, Pricing, Security, About, Blog, Help Center, Careers, Contact, Request Demo, Customer Stories, Login, Platform, KIDUORBIT, and all dynamic [slug] pages

1.5 — Background Blob Circles
Add soft blurred decorative blob circles as background elements across all sections of the website.
Reference: See attached screenshot — the testimonials section shows the exact style needed. Soft, large, blurred circles in muted brand colors placed at section edges.
Blob style rules:
Shape: border-radius: 50% — perfect circle
Effect: filter: blur(80px) to blur(120px) — very soft, no hard edges
Opacity: 0.20 – 0.35
Size: 250px – 500px
Position: absolute — placed at corners or edges of each section
pointer-events: none
z-index: 0 — behind all content
Parent section must have position: relative and overflow: hidden
Color options to use randomly:
#fcbf49 — warm yellow
#0c716b — teal
#f77f00 — orange
#ba6f20 — amber
#003049 — navy (very low opacity 0.12 only)
Placement — add blobs to ALL sections across ALL pages:
Every section gets minimum 2 blobs — one at a top corner, one at opposite bottom corner
Alternate colors between sections for visual variety
Hero sections get 3 blobs — slightly larger and slightly higher opacity (0.30 – 0.35)
Dark background sections get blobs too — use lighter color variants at lower opacity (0.15)
Random placement pattern:
Randomize per section:
Top-left + Bottom-right
Top-right + Bottom-left
Left-center + Right-center
No two adjacent sections should have blobs in the same positions.
